### Data Binding

*  It is the method which establishes a connection between the application's UI and the data it displays

* Including 3 types:

  * One-way binding: 

    syntax: string Interpolation - we use double curly braces to wrap a variable

    ```html
    <p>{{ name }}</p>
    <p>{{ object }}</p>
    // can not bind with an object directly in the template
    // the engine will convert the object to string [object object]
    
    // instead we need to specify the attribute we need
    <p>{{ object.name }}</p>
    
    
    // without the curly brace, the string will be display directly
    <p>Jack</p>
    ```



#### One - way binding:

* String Interpolation:  it is a syntax, we use double curly braces to wrap a variable, so that we can dynamically display the variable in ts file in the template as well;

  ==Component -> View==

  ```html
  <p>{{ name }}</p>
  <p>{{ object }}</p>
  // can not bind with an object directly in the template
  // the engine will convert the object to string [object object]
  
  // instead we need to specify the attribute we need
  <p>{{ object.name }}</p>
  
  
  // without the curly brace, the string will be display directly
  <p>Jack</p>
  ```

  

* [Property binding: use square brackets](#the-difference-between-property-and-attribute)

    ==Component -> View==

  * IMPORTANT: Distinguish between HTML attributes and DOM properties. While many share the smae name(e.g., id, title), some exist only as attributes. Binding to a non-existent property using [prop] will trigger a template error.
  
    ```html
    <div [innerText]="text"></div>
    <div [ngClass]="backgroundColor"></div>
    
    // custom components
    <app-user [name]="myName"></app-user>
    ```



* Event binding: enable the component to listen for and respond to user actions, such as clicks, keystrokes or mouse movement. ==View -> Component==

  * Use $event object to capture specific data from the event

    ```html
    <input (input)="handleInput($event)">
    ```

    ```typescript
    handleInput(event: Event) {
      const value = (event.target as HTMLInputElement).value;
      console.log(value)
    }
    ```

  * There are 3 kinds of event you that you can bind with DOM

  * [Standard HTML DOM Events](https://www.w3schools.com/jsref/dom_obj_event.asp)

    * Classify according to evnet object interface:
    * MouseEvent: click, dbclick, mouseenter and mouseleave(no bubbling)，mouseover a n d mouseout(event bubbling), mousemouseup, mousedown
    * KeyBoardEvent: keydown(any), keypress(only character), keyup
    * DragEvent：drag(apply to the dragged element) and dragover(the container element)(both are triggered continuesly), dragenter and drag leave
    * FocusEvent: focus (<input>, <textarea>, <select>, <a>), blur
    * TouchEvent：touchstart, touchend, touchmove, touchcancel
    * WheelEvent: wheel;
    * UiEvent: resize, scroll
    * Event: change: the content of a form element has changed vs input

  * selfdefined event implemented by @Output decorate and EventEmitter

  * Event generated by directives







##### The difference between property and attribute

* attributes are defined by the HTML source code; property are defined by the DOM

* Attributes are used to initialize DOM properties. Once the browser parses the HTML and creates the DOM nodes, the attribute's job is largely done

* Attributes are always strings, properties can be any JavaScript data type, including booleans, numbers, or objects

  * While Angular's template complier allows binding various types(Number, Boolean, String), it ultimately renders via the browser's ==*element.setAttribute('colspan', value)*== API. Since this API requires a string as its second argument, Angular automatically calls *toString()* on the value before passing it to the browser.

* Common properties (the following ones can also be used as attribute, but we prefer to use them as properties, because we can have more control in that case.):

  * disabled
  * value: Real-time content of the input box
  * src/href: image source or link address
  * innerText: used to modify the text in a tag
  * className: In angular we can also use [class]

* Common attributes:  must be used as attribute

  ```html
  // static
  <td colspan="3"></td>
  
  // dynamically
  <td [attr.colspan]="isExpanded ? 2 : 1"></td>
  <td [attr.colspan]="dynamicSpan"></td>
  ```
